## 3. Что такое "компилятор", зачем он нужен и почему некоторые языки обходятся без него?
Компилятор - программа, которая занимается тем, что переводит текст написанный на компилируемом ЯП в машинный код.
Так как процессор не может понять код который написан на языке который понятен человеку, то его с помощью компилятора переводят
в код, который будет понятен процессору. Так же ЯП могут быть интерпретируемыми, то есть для программы написанной на 
таком ЯП нужен интерпретатор, который будет покомадно переводить исходный код в команды для процессора.

## 4. Что такое "фреймворк" и для чего он нужен? Приведите примеры известных вам фреймворков.
Фреймфорк - это ПО, которое определяет структуру разрабатываемого приложения, облегчает и ускоряет разработку. Фреймворк задает
архитектурные правила по которым должно быть создано и по которым должно расширяться приложение. У фремворка есть постоянная часть
которая не будет изменяться от приложения к приложению.
Примеры фреймворков:
* React
* Vue
* Django
* Angular

## 5. Что за приставка "http://" перед адресами сайтов и почему она всё чаще теперь становится "https://"?
Эта приставка - протокол по которому общается браузер с сайтом. В данном случае - http(hyper text transfer protocol) и 
https(hyper text transfer protocol secure). Secure - потому что на все данные которые передаются по этому протоколу накладывается 
ssl шифрование. Теперь таких сайтов становиться все больше потому что компании управляющие поисковыми системами.
Которые регулируют большую часть поисковой выдачи в интернете все чаще двмгают вперед в выдаче на своих платформах те сайты, которые
могут общаться по https и опускают в выдаче сайты которые так не могут, тем самым обечпечивая безопасную передачу данных пользователей.

## 6. Самая популярная библиотека для разработки фронтенд-приложений, ReactJS, моделирует логику в виде компонентов. Если бы вам нужно было на ReactJS разработать страницу профиля ВКонтакте (например, https://vk.com/id1), на какие компоненты вы бы её разбили? Почему именно так?
* Компонент поста, который будет содержать в себе компонент комментов(состоящий из компонентов отдельных комментов). 
Таким образом легче управлять данными в компоненте + этот же компонент поста можно использовать и на других страницах.
* Компонент бокового меню которое будет использоваться на всех страницах.
* Компонент фотоленты, который можно использовать не только на странице профиля, но и на страницах групп.
* Компонент медиа(фотоальбомы, видеозаписи, музыка), который так же можно переиспользовать

## 7. SqlServer, PostgreSQL, SQLLite, MySQL, Oracle, Microsoft Access - разные базы данных с разным функционалом, которые разрабатываются, в основном, разными компаниями с разным видением своего продукта. Однако все эти базы используют один и тот же язык запросов - SQL, и не планируют от него отказываться. Как так получается? Что такого в SQL, что он подходит всем этим базам? А если он такой чудесный, то почему многие другие базы данных, вроде MongoDB или Cassandra, его не используют?
SQL - язык запросов к реляционным БД. В момент когда реляционные СУБД только появлялись были разные языки для запросов, но в итоге
было принято решение разработать единый стандарт, который будет гарантировать совместимость ПО со всеми СУБД. MongoDB и Cassandra
являются нереляционными СУБД и для доступа к данным используют NoSQL подход к запросам в БД.
